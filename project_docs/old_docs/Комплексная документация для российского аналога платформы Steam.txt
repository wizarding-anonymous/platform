## Введение

Данный документ представляет собой навигационный указатель по комплексной документации для российского аналога платформы Steam. Документация разработана с целью стандартизации и обеспечения согласованности всех компонентов микросервисной архитектуры платформы.

Особое внимание уделено интеграции микросервисов между собой, стандартизации технологического стека с учетом использования Go для бэкенда, PostgreSQL для базы данных и Flutter для фронтенда, а также обеспечению безопасности, мониторинга и готовности к продакшен-развертыванию.

## Структура документации

Документация разделена на несколько логических блоков:

1. **Анализ и аудит** - документы, содержащие результаты анализа существующих спецификаций и выявленные несоответствия
2. **Стандартизация** - документы, определяющие единые стандарты для всех компонентов системы
3. **Интеграции** - документы, описывающие взаимодействие между микросервисами и внешними системами
4. **Инфраструктура** - документы, описывающие инфраструктурные компоненты и процессы развертывания
5. **Безопасность и мониторинг** - документы, определяющие подходы к обеспечению безопасности и мониторингу системы

## Основные компоненты системы

Система состоит из следующих основных компонентов:

1. **Микросервисы бэкенда** (Go):
   - API Gateway
   - Auth Service
   - Account Service
   - Catalog Service
   - Library Service
   - Download Service
   - Payment Service
   - Social Service
   - Developer Service
   - Admin Service
   - Analytics Service
   - Notification Service

2. **Клиентские приложения** (Flutter):
   - Десктопный клиент (Windows, macOS, Linux)
   - Мобильный клиент (Android, iOS)
   - Веб-клиент

3. **Инфраструктура**:
   - PostgreSQL (основная база данных)
   - Redis (кэширование)
   - Kafka/NATS (очереди сообщений)
   - Elasticsearch (поиск и логирование)
   - Kubernetes (оркестрация)
   - Prometheus + Grafana (мониторинг)

## Ключевые изменения и обновления

В результате проведенного аудита и стандартизации были внесены следующие ключевые изменения:

1. **Технологический стек**:
   - Замена React/TypeScript на Flutter/Dart для фронтенда
   - Стандартизация версий Go (1.21+) и PostgreSQL (15+)
   - Определение единых библиотек и фреймворков для всех микросервисов

2. **Интеграции**:
   - Оптимизация API Gateway для работы с Flutter-клиентом
   - Добавление поддержки Telegram-авторизации
   - Стандартизация форматов данных и контрактов API

3. **Безопасность**:
   - Унификация подходов к аутентификации и авторизации
   - Стандартизация обработки чувствительных данных
   - Определение единых практик безопасного кодирования

4. **Мониторинг и логирование**:
   - Единый подход к структурированному логированию
   - Стандартизация метрик и алертов
   - Унификация трассировки запросов
